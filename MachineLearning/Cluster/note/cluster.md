## 具体的聚类算法

## Hierarchical算法
自顶向下和自底向上
* Agglomerative(bottom up):
  * initially, each point is a cluster
  * Repeatedly combine the two "nearest" clusters into one
    * 每次都选最近的两个cluster进行合并
    * 不断合并就能得到最上层
    * 思想很朴素简单
* Divisive(top down):
  * start with one cluster and recursively split it 
* Point assignment:
  * Maintain a set of clusters
  * Points belong to a cluster 
  * 维护一些cluster
  * 核心是不断的去合并两个最近的类
* 核心问题：
  * 1 一个类里面有多个点，如何进行表示？
  * 2 怎么定义“最近”？多个点的最近关系如何计算？
  * 3 从哪一个点开始？哪一个点结束？一定要最后结束吗？

**层次化聚类**
要么合并 要门分离
可以根据需要从某一类为截断

* 问题1
  * 如果只有一个点就好了 
  * 如果有多个点 转化为一个点
    * 非线性算法我们努力转化为线性算法 
    * 如果是欧氏空间中的点，每一个类都可以算出所有数据点的平均值，得到一个新的点的向量
    * 这个新的向量就叫做cluster的质心
    * 用质心来代表这个cluster
    * 最具有代表性
    * ** centroid **
* 问题2
  * 计算两个cluster的距离：
  * 用质心来计算两个cluster的距离

* 例子
  * 最开始每一个点都是一个cluster
  * 计算距离，合并为一个cluster，并用质心代表他
  * 可视化 用质心和半径表示
  * 欧氏距离就很方便

## 其他算法

我们的核心想法：就是把多个点的cluster转化一个带你

转化

* **clustroid** 簇心 挑出来的一个点，这个点到其他的点距离都最近
  * 什么叫做到其他店都最近呢？
  * **方法1**
    * 一种是到其它点的最大距离 最近！
      * 即 $$\min_{i} \max_{} d(x_i, x_j)$$
    * 一种是到其他点距离的平方和最近
      * 即 $$\min_{i} \sum_{j=1}^{N} d(x_j, x_i)$$
  * 簇心是算出来的一个核心点
  * 质心是算出来的虚拟点
  * **方法2**
  * 用的是代表点的方法 簇间距离：
    * 两个簇最近的两个点的最小距离
  * **方法3**
    * 尝试把两个簇合并一下，合并之后的质量高就合并，质量不高就不合并
    * 寻找合并质量最号的那个
    * 评判标准也很多
      * 簇的直径是否最小？
        * diameter
      * 簇的平均距离是否最小？
        * average distance between points
      * 使用**density-based approach**
        * 稠密度
        * 数据量和直径的一个比值

* 方法1选择代表点
* 方法2簇间距离
* 方法3凝聚评估

* 只要能找到这个点，就能进行处理多个点的情况

## K-means clustering

更主流、常用的算法 上来先尝试一下聚类分析，首选算法就是K-means算法

盲测

K-means k个均值来聚类

* 欧式空间
* k个类别
* 迭代聚类的过程：
  * 初始化 首先选择k个簇的中心点 k个簇心 
    * 初始化可以随机
    * 可以用启发式的初始化 选择k个相对比较分散的簇
      * 好处是避免一开始选的比较差，导致我的时间复杂度太高，效果太差
      * 加速算法
  * 不断迭代 最后收敛到我们想要的簇心

算法细节：
* k个类别
  * 首先给每一个点分配给离他最近的质心 完成第一轮的聚类 每一个样本都有一个子集类别的归属
  * 再每一个点都有一个类别之后，重新计算每一个cluster的质心 我们要更新
    * 因为每个点的归属可能就发生变化了
  * 再进行分类
    * 重新计算新的质心 重新分配
  * 一直到稳定了。当本轮计算结果和下一轮几乎没有变化，那么计算结束。

* 例子
  * x负半轴上有一些点，第一象限内有一些点
  * 可视化过程
  * 最后发现点的归属已经稳定下来了，质心也不会变化了
  * 这样k-Means聚类就结束了

k-Means算法比较简单，背后也有严密的数学证明，可以像感知机算法一样证明收敛性的

* 存在一个问题
  * 超参数
    * 参数：模型自己学出来的
    * 超参数：我们预设的一个值
  * k等于多少？
    * 两种情况
      * 已经知道了需要选择合适的k 例如有些问题当中k需要选择为2
      * 另一种是需要根据具体的数据进行处理
        * 4 7 9 隐含了k=3 我如何计算这个3是合适的？
        * 数据驱动 用数据本身去找到一个合适的k
        * 搜索和尝试 需要一个标准

* 评估k的最佳选择
  * 最佳的k
  * 用每一个点到它的质心的平均距离（紧凑程度）
  * 值越大散的越开
  * 值越小越集中
    * 当k越小，比如k=2，只能分成两类，这种样子平均距离就会很大
    * 当k很大，夸张一点k=N，此时每一个点自成一类，平均距离就是0
    * 所以要选择合适的k让这个平均距离-k的曲线中，挑个合适的点
    * 当曲线几乎不下降了，就选择这个k的值

前提：数据比较少

如果数据规模比较大，k-Means算法存在问题，内存不足与加载所有的数据，但是k-Means需要看到所有的数据才能进行决策

## The BFR Algorithm
大数据的最低标准：单台机器不能一次性解决

因此我们要对k-Means进行改进 处理一个大规模数据，没有办法一次性加载
* Bradely-Fayyad-Reina

### 前提假设
* 假设是标准正态分布 每一个点都是服从正态分布的 
  * axis-aligned ellipse 椭圆的长轴短轴和坐标轴平行 因为要独立
  * 只有这样才能简化计算，从局部可以看到整体
  * 把空间复杂度从 O(data) 变成了 O(cluster)
* normally distributed around a centroid in a Euclidean space
* 每次只能看到局部的数据
  * 一次加载只是提取简单的统计量
  * 核心是k-Means算法 对质心跑一下算法
  * 对中间结果划分为三个集合

* Discard set (DS)
  * 丢弃集
  * 集合中的每一个点都很明确的知道应该归属于哪一个质心
  * 不需要保留了
* Compression set (CS)
  * 压缩集
  * 很多样本是比较相近的，应该属于同一个cluster
  * 属于具体那个待定
* Retainded set (RS)
  * 保留集
  * 关系不明确 待定中

分类的依据：我把它假设服从高斯分布 我们只需记住高斯分布的参数来代表每一个cluster
* 压缩集 丢弃集 保留集都用一些统计量表示
* $N$ 点的个数
* $SUM$ 所有点向量的和向量
* $SUMSQ$ 每一个维度的平方和
这三个量可以表示高斯分布的核心参数
* 均值 $SUM_i/N$
* 方差 利用二阶矩表示 $(SUMSQ_i/N)-(SUM_i/N)^2$
* 存入这些统计量有助于我们动态的调整

方法好处很多
* 这就是假设高斯分布的好处
* 只要用均值和方差就可以表示所有的数据
* 这些参数全部只用三个统计量代替了
* 如果椭圆的坐标轴不平行，方差就是协方差，会变得很复杂
* 变成了协方差

用很少的数据量表征一个很大的数据 相当于一次特征提取 用统计量表示一类数据

* 聚类算法：无监督里面的离散情况 很多个样本 m维 聚类为k个cluster
* $x_1\cdots x_n$ 变成了一个k维向量 独热编码(one hot)
  * 每一个k维向量表示的第d个维度表示这个向量是否再d个cluster中
* 缺点：只要属于同一个cluster，向量的相似度就是0，太绝对了。我们希望有一些小小的变化，不是非0即1的
  * 可以做一些概率分布 
    * 科技、金融、金融科技
    * 金融科技可以再金融、科技都有关系

推广：软聚类
* 原来是硬聚类 一个向量只有一个分类(硬降维)
* 属于每一个cluster都有一个概率
  *
